# 브라우저 호환성 안내

이 문서는 한국어 문법 학습 웹사이트의 구형 브라우저 호환성에 관한 정보를 제공합니다.

## 구현된 호환성 솔루션

본 프로젝트는 다음과 같은 브라우저 호환성 솔루션을 구현하였습니다:

1. **JavaScript 폴리필** (`js/polyfills.js`)
   - ES6 기능 (Promise, fetch API, Array.from 등)에 대한 폴리필 제공
   - IE11 및 구형 브라우저에서 현대적인 JavaScript 기능 지원
   - 자동으로 브라우저를 감지하여 경고 메시지 표시

2. **IE 호환성 CSS** (`css/ie-compatibility.css`)
   - Flexbox 및 Grid 레이아웃에 대한 대체 스타일 제공
   - 미디어 쿼리를 통한 반응형 레이아웃 지원
   - 인쇄 시 최적화된 스타일 적용

3. **HTML 표준화**
   - 모든 페이지에 HTML5 Doctype 추가
   - 반응형 웹을 위한 viewport 메타 태그 추가
   - 구형 브라우저에서도 올바르게 렌더링되도록 구조 최적화

## 지원 브라우저

이 웹사이트는 다음 브라우저에서 최적화되어 있습니다:

- **Chrome**: 버전 60 이상 (권장)
- **Firefox**: 버전 60 이상
- **Safari**: 버전 10 이상
- **Edge**: 버전 16 이상
- **Internet Explorer**: 버전 11 (기본 기능 지원)

## 구형 브라우저 사용자를 위한 안내

Internet Explorer 11 이하 버전을 사용하시는 경우:

1. 일부 최신 시각적 효과가 제한될 수 있습니다.
2. 페이지 로딩 속도가 최신 브라우저보다 느릴 수 있습니다.
3. 최적의 경험을 위해 Chrome, Firefox, Edge와 같은 최신 브라우저로 업그레이드하는 것을 권장합니다.

## 호환성 스크립트 실행 방법

프로젝트에 브라우저 호환성 솔루션을 적용하려면 다음 명령을 실행하세요:

```
node add_polyfill.js
```

이 스크립트는 다음 작업을 수행합니다:

- 모든 HTML 파일에 폴리필 스크립트 참조 추가
- 모든 HTML 파일에 IE 호환성 CSS 참조 추가
- HTML5 Doctype 확인 및 추가
- viewport 메타 태그 확인 및 추가

## 추가 개발자 정보

개발 시 다음 사항을 고려해주세요:

1. **JavaScript 작성 시**
   - ES6 기능 사용 시 폴리필 의존성을 인지할 것
   - 가능하면 IE 호환 코드를 작성하되, 불가능한 경우 적절한 polyfill.js 사용

2. **CSS 작성 시**
   - Flexbox와 Grid는 신중하게 사용하고 IE 호환성을 고려할 것
   - 미디어 쿼리 기준점으로 768px 사용 (표준화됨)
   - 가능한 경우 퍼센트 기반 레이아웃 사용

3. **HTML 구조**
   - 시맨틱 요소 사용 시 IE 지원 고려
   - 최신 HTML5 요소 사용 시 적절한 대체 구조 제공

## 문제 해결

구형 브라우저 호환성 관련 문제가 발생하면 다음을 확인하세요:

1. HTML 파일에 폴리필 스크립트와 IE 호환성 CSS가 올바르게 포함되어 있는지 확인
2. 복잡한 CSS 레이아웃이 IE11에서 올바르게 렌더링되는지 확인
3. JavaScript 오류가 발생하는 경우 해당 기능에 대한 폴리필이 있는지 확인
